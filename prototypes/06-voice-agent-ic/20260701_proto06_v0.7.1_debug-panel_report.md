# Prototype 06 — Rapport V0.7.1

## Objectif de l'itération

La V0.7 introduit des variantes discursives via `testimonyVariantCatalog`.

La V0.7.1 ajoute une mini infrastructure de debug pédagogique, destinée au concepteur du prototype. Elle permet de vérifier quelle variante est réellement utilisée selon le personnage et le scénario pédagogique choisi.

Le debug est invisible par défaut et ne modifie pas l'expérience utilisateur normale.

## Activation et désactivation

Activation depuis la console navigateur :

```js
localStorage.setItem("proto06Debug", "1")
location.reload()
```

Désactivation :

```js
localStorage.removeItem("proto06Debug")
location.reload()
```

Raccourci ajouté :

```js
window.proto06Debug(true)
window.proto06Debug(false)
```

## Champs affichés dans le lecteur

Quand le debug est actif, un panneau flottant repliable apparaît en bas à droite.

Il affiche :

* scénario pédagogique actif ;
* personnages sélectionnés ;
* personnage actuellement actif ;
* langue active ;
* `speechLang` demandé ;
* lieu / pays ;
* texte original du témoignage actif ;
* traduction française ;
* mots transparents ;
* tags discursifs ;
* fallback utilisé ou non ;
* source du fallback ;
* index de l'étape active ;
* question commune.

## Traductions françaises

`data-0.0.7.1.js` ajoute `testimonyTranslationFrCatalog`, puis enrichit les variantes existantes avec un champ `translationFr`.

Les traductions couvrent les sept personnages et les quatre scénarios disposant de variantes dédiées :

* `globalUnderstanding`
* `affectsFeelings`
* `adaptationStrategies`
* `prospectiveStories`

Ces traductions sont destinées au concepteur/testeur et n'apparaissent que dans le panneau debug.

* section technique conditionnelle dans l'aperçu

## Debug dans le compositeur

Quand `proto06Debug` est actif, le compositeur affiche une section technique dans l'aperçu :

* id du scénario pédagogique ;
* ids des personnages sélectionnés ;
* nombre de variantes disponibles ;
* variantes manquantes pour le scénario choisi.

Cette section reste invisible quand le debug est désactivé.

## Limites restantes

* Le panneau debug est un outil de conception, pas une interface d'analyse complète.
* Les traductions françaises sont manuelles.
* Les scénarios `transparentWords` et `clarificationZones` continuent de réutiliser les variantes globales.
* La vérification visuelle du panneau reste à faire dans plusieurs tailles d'écran.

## Vérifications effectuées

* debug désactivé par défaut via absence de `proto06Debug` ;
* activation prévue par `localStorage.setItem("proto06Debug", "1")` ;
* désactivation prévue par `localStorage.removeItem("proto06Debug")` ;
* raccourci `window.proto06Debug(true/false)` ajouté ;
* traduction française exposée dans l'état debug ;
* mise à jour du panneau lors du changement de personnage actif ;
* aucune création du panneau quand le debug est désactivé ;
* `node --check data-0.0.7.1.js` ;
* `node --check composer-0.0.7.1.js` ;
* `node --check script-0.0.7.1.js`.
