# IC-Hub V0.8.3 - AI admin onboarding

Date: 2026-07-03

## Objectif

La V0.8.3 ajoute une couche de tableau de bord decisionnel au-dessus du catalogue IA. Elle vise une personne qui reprend le projet et doit comprendre rapidement ce qui est actif, ce qui est seulement prepare, et quelles actions sont prudentes.

Cette passe n'active aucune generation IA.

## Changements principaux

- Creation de `admin-0.8.3.html`, `admin-0.8.3.css`, `admin-0.8.3.js`.
- Creation de `hub-0.8.3.html`, `hub-0.8.3.css`, `hub-0.8.3.js`.
- Redirection `/admin.html` vers `admin-0.8.3.html`.
- Redirection `/hub.html` vers `hub-0.8.3.html`.
- Version serveur/package passee a `0.8.3`.
- Creation du guide durable `docs/ai-admin-maintainer-guide.md`.

La V0.8.2 reste disponible dans ses fichiers propres.

## Bandeau d'etat IA

La section IA affiche maintenant clairement :

- aucune IA generative n'est active dans les parcours apprenants;
- Prototype 06 utilise le runtime scenarise;
- les cles ne sont pas visibles cote navigateur;
- le navigateur n'appelle pas OpenAI.

## Checklist administrateur

La checklist calcule ou rappelle :

- `.env` ignore par Git;
- cle OpenAI detectee cote serveur oui/non;
- provider OpenAI actif oui/non;
- derniere synchronisation connue;
- nombre de modeles au catalogue;
- nombre de modeles recommandes IC-Lab;
- nombre de modeles autorises runtime;
- nombre de modeles avec prix renseigne;
- Prototype 06 generatif : non.

Aucun secret n'est affiche.

## Prochaine etape recommandee

La logique V0.8.3 reste simple :

- aucun modele : synchroniser;
- modeles sans recommandation : marquer 2 ou 3 modeles recommended;
- recommandations sans AIConfig liee : associer une AIConfig a un modele catalogue;
- sinon : creer un mode de test serveur controle, sans flux apprenant.

## Modeles recommandes

Un bloc `Modeles recommandes pour commencer` affiche jusqu'a 5 modeles selon :

- `icLabRecommendation = recommended`;
- `allowedForRuntime = true`;
- `modality = text`;
- cout `low` ou `medium`;
- statut `available`.

Chaque carte explique l'usage, le cout, le prix et pourquoi le modele est propose. Un bouton permet de le retrouver dans le catalogue.

## A eviter

La section `A eviter pour l'instant` resume les familles a manipuler avec prudence :

- realtime;
- image;
- embeddings;
- moderation;
- stale/deprecated.

## Glossaire et reprise projet

La V0.8.3 ajoute :

- un glossaire rapide;
- un bloc `Si vous reprenez ce projet`;
- une procedure courte de reprise;
- un guide mainteneur plus durable dans `docs/ai-admin-maintainer-guide.md`.

## Prudence sur les actions

Des messages rappellent que :

- synchroniser OpenAI contacte OpenAI cote serveur uniquement pour lire la liste des modeles;
- modifier un provider modifie les metadonnees locales, pas le compte provider;
- autoriser runtime ne suffit pas a activer l'IA dans Proto06;
- les prix doivent etre verifies sur les tarifs officiels.
