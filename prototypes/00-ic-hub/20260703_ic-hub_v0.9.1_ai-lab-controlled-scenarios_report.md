# IC-Hub V0.9.1 - AI Lab scenarios controles

Date: 2026-07-03

## Objectif

La V0.9.1 transforme l'AI Lab V0.9 en banc d'essai pedagogique admin-only. Elle permet de generer et comparer des brouillons IC controles sans brancher Prototype 06, sans modifier le parcours apprenant et sans stocker les textes generes.

## Formulaire de generation controlee

L'admin V0.9.1 ajoute une section `AI Lab - scenarios controles` avec choix explicites :

- modele OpenAI autorise depuis le catalogue local;
- personnage;
- langue cible;
- scenario pedagogique;
- theme climatique limite a 120 caracteres;
- longueur cible;
- niveau de prudence pedagogique.

Le frontend n'envoie jamais de prompt complet. Il envoie seulement des champs controles a l'endpoint serveur.

## Validations serveur

`POST /api/admin/ai/runtime/ic-agent-draft` valide :

- modele du catalogue local seulement;
- provider `openai`;
- `allowedForRuntime = true`;
- statut `available` ou `active`;
- modele non stale/deprecated;
- `providerModelId` present;
- personnage autorise;
- langue autorisee;
- scenario autorise;
- `maxWords` borne entre 20 et 60;
- `pedagogicalStrictness` dans `safe` ou `exploratory`;
- theme climatique present et limite a 120 caracteres.

L'endpoint refuse aussi les champs de prompt libre, texte apprenant, launchToken, token, email, userId ou donnees personnelles courantes.

## Sortie structuree

La generation retourne maintenant :

- `draft.text`;
- personnage, langue, scenario et theme;
- nombre de mots;
- `checks` locaux simples;
- `safety` explicite;
- `stored: false`;
- `runtimeId: admin_openai_ic_agent_draft`.

Les checks restent volontairement simples : markdown, point d'interrogation, longueur approximative et langue cible attendue.

## Brouillons de session

Les brouillons generes sont conserves uniquement en memoire JavaScript de la page admin :

- affichage compare par brouillon;
- metadonnees pedagogiques;
- nombre de mots;
- checks simples;
- bouton copier;
- bouton supprimer;
- bouton vider la session.

Il n'y a pas de `localStorage`, pas d'ecriture DB et pas d'ecriture `run_events`. Un rechargement de page vide les brouillons.

## Clara / Marco / Ana

L'admin garde les boutons rapides :

- Generer Clara;
- Generer Marco;
- Generer Ana;
- Generer les 3 personnages principaux.

Le bouton des trois personnages lance trois appels successifs, pas paralleles, pour limiter la pression sur l'API.

## Securite et non-branchement Prototype 06

- `OPENAI_API_KEY` reste cote serveur.
- `.env` n'est jamais affiche ni logge.
- Le navigateur n'appelle jamais OpenAI directement.
- Les appels OpenAI utilisent `store: false`.
- Aucun texte apprenant n'est envoye.
- Aucun launchToken n'est accepte.
- Aucun brouillon n'est stocke en base.
- Aucun texte genere n'est stocke dans `run_events`.
- Prototype 06 n'est pas modifie.
- Aucun runtime generatif n'est active dans le parcours apprenant.

## Limites restantes

- Les checks pedagogiques sont locaux et simples.
- La detection effective de langue n'est pas ajoutee dans cette passe.
- Les couts restent qualitatifs quand les prix ne sont pas renseignes dans le catalogue.
- Les brouillons ne sont pas historises volontairement.
- L'AI Lab ne valide pas encore une activite complete.

## Prochaine etape possible

Ajouter une grille d'evaluation admin non stockante pour scorer manuellement les brouillons : clarte IC, transparence lexicale, adequation scenario, prudence pedagogique et cout estime. Cette etape peut rester hors Prototype 06.
