# Prototype 06 V1.2 - squelette Agent Runtime

Date : 2026-07-02

## Objectif

Preparer l'integration future de modes IA/agents dans Prototype 06 sans appeler de provider externe et sans changer le comportement pedagogique actuel.

## Fichiers crees

- `index-1.2.html`
- `script-1.2.js`
- `style-1.2.css`

## Separation introduite

La V1.2 separe plus clairement :

- scenario pedagogique ;
- participants ;
- textes scenarises ;
- runtime agent ;
- configuration IA assignee ;
- affichage ;
- traces hub.

## Runtime scenarise

`scriptedAgentRuntime` devient la source officielle des tours de parole actuels.

Il expose :

- `id`
- `label`
- `runtimeEnabled`
- `canGenerateText`
- `canGenerateAudio`
- `resolveTurn(context)`
- `describeRuntime(aiConfig)`

Les temoignages, relances pedagogiques et voix navigateur restent ceux des versions precedentes. Aucun texte n'est genere dynamiquement.

## Runtimes futurs desactives

Un runtime generique `disabledFutureRuntime` represente les modes preparatoires :

- IA texte future + voix navigateur ;
- IA texte future + voix API ;
- agent vocal temps reel ;
- multi-agents experimental.

Ces modes ne lancent aucune API, ne generent aucun texte, ne generent aucun audio et retombent automatiquement sur `scriptedAgentRuntime`.

## Selection runtime

`selectAgentRuntime(aiConfig)` applique une politique sure :

- aucune config : runtime scenarise ;
- `runtimeEnabled !== true` : runtime scenarise ;
- statut non `active` : runtime scenarise avec warning ;
- mode `scripted` : runtime scenarise ;
- autre mode : fallback scenarise avec warning.

En V1.2, le runtime reellement utilise reste toujours `scripted`.

## Affichage enseignant

La carte enseignante affiche maintenant :

- runtime utilise ;
- configuration assignee ;
- IA reelle non activee ;
- fallback oui/non ;
- warnings si le mode assigne est preparatoire.

## Debug

Le debug affiche :

- `aiConfigId` ;
- config assignee ;
- runtime selectionne ;
- `runtimeEnabled` ;
- fallback runtime ;
- dernier tour : source, `generated`, `runtimeId` ;
- warnings runtime.

## Events Hub

Prototype 06 V1.2 tente d'envoyer des events non sensibles :

- `ai_config_resolved`
- `agent_runtime_selected`

Payloads filtres :

- identifiant de config ;
- statut ;
- mode ;
- runtime selectionne ;
- fallback ;
- `generated=false`.

Aucun `launchToken`, aucune cle, aucun texte apprenant complet et aucun texte genere ne sont envoyes.

## Absence d'IA reelle

La V1.2 ne contient :

- aucun appel OpenAI ;
- aucun appel provider externe ;
- aucune cle API ;
- aucun secret ;
- aucune generation de texte ;
- aucune generation audio IA.

Les seuls `fetch()` ajoutent ou conservent des appels locaux vers le Hub `127.0.0.1:8790`.

## Limites restantes

- Le fallback vers le runtime scenarise est volontairement systematique pour tout mode IA non encore active.
- La verification fine des interactions vocales reste manuelle dans le navigateur.
- Les modes IA futurs n'ont pas encore d'implementation reelle.
