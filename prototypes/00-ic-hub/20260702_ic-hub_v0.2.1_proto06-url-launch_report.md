# IC-Lab Hub V0.2.1 - URL de lancement Prototype 06 V1.1.1

## Objectif

Cette micro-passe met a jour le hub pour utiliser la micro-version Prototype 06 V1.1.1, capable de lire les parametres URL `activityId`, `activitySource`, `courseId` et `assignmentId`.

## Fichiers modifies

- `server/data/prototypes.json`
- `server/server.js`
- `public/student-0.2.js`
- `public/teacher-0.2.js`

## Ancienne URL

```txt
http://127.0.0.1:8788/index-1.1.html
```

## Nouvelle URL

```txt
http://127.0.0.1:8788/index-1.1.1.html
```

## Parametres transmis

Les liens de lancement Prototype 06 generes par le hub transmettent maintenant :

- `activityId`
- `activitySource`
- `courseId` si present dans l'assignation
- `assignmentId` si present dans l'assignation

Exemple :

```txt
http://127.0.0.1:8788/index-1.1.1.html?activityId=climateObservations001&activitySource=server&courseId=course_demo_repli4c&assignmentId=assign_demo_001
```

## Student

`student-0.2.js` utilise `proto.launchUrl` et ajoute les quatre parametres possibles au bouton `Lancer`.

## Teacher

`teacher-0.2.js` ajoute aussi un lien `Lancer` dans la liste des activites assignees, utile pour verifier rapidement une assignation depuis l'espace enseignant.

## Verifications effectuees

- `node --check server.js`
- `node --check public/student-0.2.js`
- `node --check public/teacher-0.2.js`
- verification du connecteur `proto06`
- verification qu'une assignation produit une URL vers `index-1.1.1.html`
- verification que l'URL contient `activityId`, `activitySource`, `courseId` et `assignmentId`

## Limite

Cette passe ne modifie pas Prototype 06. Elle suppose que `index-1.1.1.html` est disponible sur le backend Prototype 06 local.
