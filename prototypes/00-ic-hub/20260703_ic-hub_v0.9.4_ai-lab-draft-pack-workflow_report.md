# IC-Hub V0.9.4 - AI Lab draft pack workflow

Date: 2026-07-03

## Objectif

La V0.9.4 ajoute un format de lot commun de brouillons a evaluer. Un admin peut generer un lot, l'exporter sans notes, puis plusieurs evaluateurs peuvent importer exactement ce meme lot, l'evaluer localement et exporter leurs evaluations. La comparaison multi-evaluateurs devient ainsi comparable brouillon par brouillon.

## Draft pack

Nouveau bouton : `Exporter le lot de brouillons a evaluer`.

Schema exporte :

- `schema: ic-hub-ai-lab-draft-pack`;
- `schemaVersion: 0.9.4`;
- `createdAt`;
- `draftPackId`;
- `drafts[]`;
- `draftId`;
- `text`;
- `metadata`;
- `checks`;
- `safety`.

Le draft pack ne contient pas de notes, pas de token, pas de cle, pas de donnee apprenante.

## Import draft pack

La section `Importer un lot de brouillons a evaluer` :

- importe cote navigateur uniquement;
- valide le schema;
- limite les fichiers a 2 Mo;
- remplace les brouillons de session par ceux du pack;
- rend la grille d'evaluation disponible sans regeneration OpenAI.

## Export evaluation lie au pack

L'export d'evaluation V0.9.4 inclut :

- `draftPackId`;
- `draftPackIds`;
- `evaluator.label`;
- `ratings`;
- `score`;
- `comment`;
- `favorite`;
- `safety`;
- les `draftId` originaux.

## Comparaison multi-evaluateurs

La comparaison groupe maintenant les evaluations par :

`draftPackId + draftId`

L'interface rappelle d'importer des exports issus du meme draft pack. Si plusieurs `draftPackId` sont detectes, la synthese affiche un warning.

## Pack reconstruit depuis comparaison

Un export de comparaison peut etre importe comme draft pack de secours. L'interface affiche :

`Format reconstruit depuis comparaison, a utiliser seulement pour test.`

Ce chemin sert uniquement a tester ou recuperer des textes depuis `aggregates`.

## Absence de stockage serveur

La V0.9.4 ne stocke rien cote serveur :

- pas de draft pack en base;
- pas d'evaluation en base;
- pas d'import en base;
- pas de `run_events`;
- pas de `localStorage` pour brouillons, packs, evaluations ou comparaisons.

## Absence de branchement Prototype 06

Prototype 06 n'est pas modifie. Le runtime apprenant reste scenarise. Aucun brouillon n'est injecte dans une activite.

## Limites restantes

- Le workflow repose sur des fichiers JSON manuels.
- Aucun chiffrement d'export n'est ajoute.
- Les evaluateurs doivent recevoir le meme draft pack pour une comparaison propre.
- La reconstruction depuis comparaison est uniquement un secours de test.
