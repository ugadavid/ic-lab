# IC-Lab Hub V0.2 - connecteur Prototype 06

## Objectif

La V0.2 prouve que le hub IC-Lab peut orchestrer un prototype externe, ici Prototype 06, sans absorber sa logique interne. Un enseignant peut demander au hub la liste des activites serveur exposees par Prototype 06 et assigner une activite a un cours.

## Pourquoi un connecteur

Le hub ne copie pas les donnees internes du Prototype 06. Il conserve seulement une configuration de connexion, interroge son API locale et stocke l'assignation cote cours. Prototype 06 reste responsable de sa bibliotheque, de son lecteur et de son backend.

## Endpoint ajoute

- `GET /api/prototypes/:prototypeId/activities`

Pour `proto06`, le hub appelle `http://127.0.0.1:8788/api/activities` et retourne une enveloppe avec `prototypeId`, `source`, `connectorStatus` et `activities`.

Un endpoint admin a aussi ete ajoute :

- `GET /api/prototypes/status`

## Prototype 06 disponible

Si le serveur Prototype 06 repond, le hub retourne `connectorStatus: "ok"` et la liste des activites serveur. Chaque activite est conservee comme reference externe.

## Prototype 06 indisponible

Si le serveur `8788` ne repond pas, le hub retourne `connectorStatus: "unavailable"`, un message lisible et une liste vide. Les cours, les prototypes et les assignations existantes restent visibles.

## Teacher V0.2

`teacher-0.2.html/js/css` ajoute une vraie selection d'activites :

- choix du cours ;
- choix du prototype ;
- chargement des activites si le prototype les supporte ;
- cartes compactes avec titre, objectif, scenario, duree, personnages, source et id ;
- fallback manuel si aucune activite n'est disponible.

## Student V0.2

`student-0.2.html/js/css` affiche les assignations de facon plus lisible avec le titre, l'objectif si disponible et un bouton `Lancer`.

Pour Prototype 06, le lien genere :

```txt
http://127.0.0.1:8788/index-1.1.html?activityId=...&activitySource=server
```

La prise en compte complete de ces parametres reste une prochaine passe cote Prototype 06.

## Stockage enrichi

Les assignations dans `server/data/course-activities.json` acceptent maintenant :

- `activityTitle`
- `activitySnapshot`

Le snapshot minimal permet d'afficher une activite deja assignee meme si Prototype 06 est temporairement indisponible.

## Limites restantes

- Le hub ne lit que les activites serveur de Prototype 06, pas les activites natives du catalogue frontend.
- Le lancement par URL doit encore etre pris en charge cote Prototype 06.
- Les connecteurs restent locaux et experimentaux.
- Le stockage reste JSON local.
