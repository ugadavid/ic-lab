# IC-Lab Hub V0.3 - Runs et traces serveur

Date : 2026-07-02

## Objectif

Ajouter une première trace de session entre le hub IC-Lab et Prototype 06, sans changer l'architecture du hub : stockage JSON local, serveur Node existant sur le port 8790, pas de base de données, pas d'IA et pas d'authentification plus complexe.

## Protections ajoutees

- Nouveau store local `runs.json` initialise au demarrage si absent.
- Endpoint `POST /api/runs/start` reserve au role `student`.
- Verification que l'etudiant a acces au cours et que l'assignation existe dans ce cours.
- Generation d'un `runId` et d'un `launchToken`; seul le hash SHA-256 du token est conserve.
- Endpoint `POST /api/runs/:runId/events` protege par `launchToken`, sans auth Bearer classique.
- Liste stricte de types d'evenements acceptes.
- Sanitisation de `user_answer_submitted` : stockage de `answerLength`, `hasText` et `timestamp`, sans texte apprenant complet.
- Mise a jour du statut de session : `created`, `launched`, `started`, `completed`, `error`.

## URL de lancement

Ancienne URL Prototype 06 :

`http://127.0.0.1:8788/index-1.1.1.html`

Nouvelle URL Prototype 06 :

`http://127.0.0.1:8788/index-1.1.2.html`

Parametres transmis :

- `activityId`
- `activitySource`
- `courseId`
- `assignmentId`
- `runId`
- `launchToken`

## Interfaces

- `student-0.3.js` remplace le lien direct par un bouton qui appelle `/api/runs/start`, puis ouvre `launchUrl`.
- `teacher-0.3.js` affiche une section `Traces recentes` avec activite, etudiant, statut, date de lancement, duree et nombre d'evenements.
- Les pages V0.2 restent presentes et non modifiees.

## Limites restantes

- Les traces restent locales au fichier JSON et ne sont pas adaptees a un usage multi-process.
- Le token est passe dans l'URL de lancement, ce qui convient au prototype local mais pas a une mise en production.
- Les pages V0.3 restent volontairement proches de V0.2 : pas de tableau avance ni d'export.
