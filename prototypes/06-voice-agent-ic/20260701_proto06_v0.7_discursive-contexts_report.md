# Prototype 06 — Rapport V0.7

## Objectif de l'itération

La V0.6 permettait déjà de choisir un scénario pédagogique, mais les personnages prononçaient toujours le même témoignage.

La V0.7 ajoute une couche expérimentale de contextes discursifs : un même personnage peut maintenant avoir plusieurs variantes de témoignage selon l'activité choisie.

## Structure de données retenue

Les informations stables restent dans les catalogues existants :

* langue dans `languageCatalog` ;
* lieu dans `placeCatalog` ;
* image, rôle et bio dans `characterCatalog` ;
* scénario pédagogique dans `pedagogicalScenarioCatalog`.

La V0.7 ajoute `testimonyVariantCatalog`.

Cette structure associe :

* un `characterId` ;
* un `pedagogicalScenarioId` ;
* un texte de témoignage ;
* des mots transparents ;
* quelques tags discursifs.

Exemple de logique :

```js
testimonyVariantCatalog.clara.affectsFeelings.text
```

## Scénarios avec variantes

Des variantes ont été ajoutées pour les sept personnages disponibles :

* Clara ;
* Marco ;
* Ana ;
* Malik ;
* Lucía ;
* Andrei ;
* Laia.

Quatre scénarios disposent d'une variante dédiée :

* `globalUnderstanding` — constat climatique ;
* `affectsFeelings` — émotion, inquiétude ou ressenti ;
* `adaptationStrategies` — réponse possible ou habitude qui change ;
* `prospectiveStories` — projection dans dix ou vingt ans.

Les scénarios `transparentWords` et `clarificationZones` réutilisent pour l'instant la variante `globalUnderstanding`.

## Fonctionnement du fallback

La fonction `getTestimonyForCharacter(characterId, pedagogicalScenarioId)` applique l'ordre suivant :

1. chercher une variante exacte pour le personnage et le scénario pédagogique ;
2. sinon utiliser la variante `globalUnderstanding` du personnage ;
3. sinon utiliser l'ancien témoignage présent dans `scenarioCatalog` ;
4. sinon afficher un placeholder clair.

Cette logique permet d'ajouter progressivement des variantes sans bloquer le prototype.

## Compositeur

Le compositeur V0.7 conserve le fonctionnement de la V0.6 :

* choix du scénario pédagogique ;
* choix de 2 à 4 personnages ;
* aperçu ;
* lancement via `localStorage`.

L'aperçu indique maintenant le contexte discursif choisi et affiche les extraits qui seront utilisés pour les personnages sélectionnés.

## Rencontre générée

La rencontre V0.7 continue d'adapter :

* la consigne ;
* la question commune ;
* l'Observation IC ;
* les relances pédagogiques.

Elle adapte maintenant aussi les témoignages eux-mêmes, selon le scénario pédagogique sélectionné.

## Limites restantes

* Les variantes sont courtes et encore scénarisées manuellement.
* Seuls quatre scénarios ont des variantes dédiées.
* Les scénarios de mots transparents et de clarification réutilisent les témoignages globaux.
* La qualité vocale dépend toujours des voix disponibles dans le navigateur.
* La scène collective reste pensée pour une rencontre courte de 2 à 4 participants.
