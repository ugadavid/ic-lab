# Prototype 06 — Rapport V0.7.2

## Objectif de l'itération

La V0.7.1 a introduit un panneau debug pédagogique fonctionnel, mais trop compact pour les tests.

La V0.7.2 améliore uniquement l'ergonomie du debug : panneau plus large, lecture en deux colonnes, traduction française plus visible et boutons de reset ciblés.

L'expérience utilisateur normale reste inchangée lorsque le debug est désactivé.

## Nouvelle disposition du panneau debug

Le panneau debug reste flottant, repliable et invisible par défaut.

Quand il est actif, sa largeur passe à :

```css
width: min(900px, calc(100vw - 2rem));
```

Le contenu dispose d'un scroll interne avec une hauteur maximale d'environ `55vh`.

## Colonne gauche

La colonne gauche affiche le contexte technique et pédagogique :

* scénario pédagogique actif ;
* titre du scénario ;
* personnages sélectionnés ;
* personnage actif ;
* langue active ;
* `speechLang` demandé ;
* lieu / pays ;
* index de l'étape active ;
* fallback utilisé ;
* source du fallback ;
* mots transparents ;
* tags discursifs ;
* question commune.

## Colonne droite

La colonne droite est dédiée au texte :

* texte original du témoignage actif ;
* traduction française du témoignage actif.

La traduction française est affichée avec un corps plus grand et une interligne plus confortable pour faciliter la lecture pendant les tests.

## Boutons de reset

Deux boutons ont été ajoutés dans le panneau debug :

* `Reset rencontre`
* `Désactiver debug`

`Reset rencontre` supprime les clés locales `proto06*` en conservant `proto06Debug`, puis recharge la page.

`Désactiver debug` supprime aussi `proto06Debug`, puis recharge la page.

## Clés localStorage supprimées

La fonction `clearProto06LocalStorage()` supprime uniquement les clés dont le nom commence par :

```js
proto06
```

Elle n'utilise pas `localStorage.clear()`, afin d'éviter de supprimer des données sans rapport avec le prototype.

## Compositeur

La section debug du compositeur a été légèrement améliorée :

* ids plus lisibles ;
* scénario choisi explicite ;
* personnages sélectionnés explicites ;
* variantes disponibles ;
* variantes manquantes ;
* boutons `Reset rencontre` et `Désactiver debug`.

Cette section reste invisible si `proto06Debug` n'est pas actif.

## Fonctions ajoutées

Dans `script-0.0.7.2.js` et `composer-0.0.7.2.js` :

* `clearProto06LocalStorage({ keepDebug })`
* `resetProto06State({ keepDebug })`

L'activation existante est conservée :

```js
localStorage.setItem("proto06Debug", "1")
window.proto06Debug(true)
```

## Limites restantes

* La vérification visuelle fine du panneau reste à faire dans le navigateur.
* Le panneau est responsive, mais son usage optimal reste desktop/tablette.
* Le reset recharge la page actuelle plutôt que de rediriger systématiquement vers le compositeur.
