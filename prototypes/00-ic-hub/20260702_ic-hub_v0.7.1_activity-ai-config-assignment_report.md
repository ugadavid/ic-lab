# IC-Lab Hub V0.7.1 - Association AIConfig aux assignations

Date : 2026-07-02

## Objectif

La V0.7.1 permet d'associer une configuration IA/voix a une activite assignee a un cours, sans activer de vraie IA.

L'enseignant peut choisir un mode pour une assignation Prototype 06 :

- scenarise + voix navigateur;
- IA texte future + voix navigateur;
- IA texte future + voix API;
- vocal temps reel futur;
- multi-agents experimental.

## Pourquoi au niveau de l'assignation

Une meme activite globale peut etre utilisee dans plusieurs cours avec des intentions differentes. Le choix du mode IA appartient donc d'abord a l'usage pedagogique dans un cours, pas a l'activite referencee globale.

La V0.7.1 stocke ce choix dans `course_activities.ai_config_id`. La table `activity_ai_config` reste disponible pour une evolution future plus fine.

## Mode par defaut

Pour Prototype 06, le mode par defaut est :

`aicfg_proto06_scripted_browser_voice`

Il correspond au mode sans IA, scenarise, voix navigateur, gratuit et actif.

## Changements donnees

- Ajout de `ai_config_id` a `course_activities`.
- Migration douce cote Node pour donner le mode par defaut aux anciennes assignations Proto06.
- Migration MariaDB qui alimente `ai_config_id` depuis les stores JSON.

## Procedure stockee

`sp_assign_activity_with_ownership` accepte maintenant `p_ai_config_id` et stocke la valeur dans `course_activities.ai_config_id`.

Les procedures stockees V0.6.2 restent conservees :

- `sp_create_run`;
- `sp_append_run_event`;
- `sp_assign_activity_with_ownership`.

## API

Endpoints ajoutes :

- `GET /api/course-activities/:assignmentId/ai-config`
- `PUT /api/course-activities/:assignmentId/ai-config`

Regles :

- teacher proprietaire du cours ou admin peut modifier;
- utilisateur ayant acces au cours peut lire;
- student lit aussi indirectement via les activites assignees.

Les reponses existantes de `GET /api/courses/:courseId/activities` incluent maintenant `aiConfigId` et `aiConfig`.
