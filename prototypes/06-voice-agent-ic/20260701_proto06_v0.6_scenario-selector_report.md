# Prototype 06 — Rapport V0.6

## Objectif de l'itération

La V0.6 ajoute une nouvelle couche pédagogique au Prototype 06 : le choix du scénario d'activité.

La V0.5.1 composait déjà une rencontre à partir de personnages, langues et lieux. La V0.6 montre qu'un même groupe de personnes peut être utilisé pour plusieurs activités d'intercompréhension : compréhension globale, ressentis, adaptation, projection, repérage d'indices ou clarification.

## Scénarios ajoutés

La V0.6 ajoute `pedagogicalScenarioCatalog` dans `data-0.0.6.js`.

Six scénarios pédagogiques sont disponibles :

* `globalUnderstanding` — Compréhension globale.
* `affectsFeelings` — Affects et ressentis.
* `adaptationStrategies` — Stratégies d'adaptation.
* `prospectiveStories` — Récits prospectifs.
* `transparentWords` — Mots transparents et indices.
* `clarificationZones` — Zones floues et clarification.

Chaque scénario contient :

* un titre ;
* une description ;
* un type de tâche ;
* une question commune ;
* un focus d'observation ;
* un nombre de participants recommandé ;
* une difficulté ;
* une consigne ;
* une observation IC initiale adaptée.

## Fonctionnement du sélecteur

Le compositeur V0.6 ajoute une première étape :

1. Choisir un scénario pédagogique.
2. Choisir 2 à 4 personnages.
3. Prévisualiser la rencontre.
4. Lancer la rencontre.

Les cartes de scénarios affichent le titre, la description, le type de tâche, la difficulté, le nombre de participants recommandé et les focus d'observation.

Le scénario choisi apparaît dans l'aperçu de la rencontre et dans l'objet de configuration généré.

## Stockage local

Au lancement, le compositeur stocke :

* `proto06SelectedCharacters` pour les personnages ;
* `proto06SelectedPedagogicalScenario` pour le scénario pédagogique.

La rencontre V0.6 lit ces deux valeurs dans `localStorage`.

## Articulation personnages / scénario pédagogique

Les personnages restent issus de `characterCatalog`.

Les langues restent issues de `languageCatalog`.

Les lieux restent issus de `placeCatalog`.

Les témoignages restent issus de `scenarioCatalog`.

Le scénario pédagogique vient de `pedagogicalScenarioCatalog` et modifie la tâche demandée à l'utilisateur sans réécrire les témoignages.

## Ce qui change dans la rencontre générée

Dans `script-0.0.6.js`, la fonction `buildScenarioFromPedagogicalSelection()` reconstruit la rencontre à partir :

* des personnages sélectionnés ;
* du scénario pédagogique sélectionné.

Le scénario choisi modifie :

* la consigne affichée ;
* la question commune ;
* l'Observation IC initiale ;
* les mots-clés/focus utilisés pour la relance pédagogique ;
* le statut pédagogique visible dans la configuration du compositeur.

La page de rencontre ajoute aussi un lien de retour vers `composer-0.0.6.html`.

## Limites actuelles

* Les témoignages ne changent pas encore selon le scénario pédagogique.
* Les relances restent scénarisées de manière simple.
* Le compositeur ne recommande pas encore automatiquement les meilleurs personnages selon l'activité.
* La scène collective reste limitée à une composition acceptable de 2 à 4 participants.
* Il n'y a pas encore de sauvegarde nommée d'une rencontre composée.
* Le stockage reste local au navigateur.

## Prochaines pistes

* Créer des variantes de témoignages selon les scénarios pédagogiques.
* Ajouter des recommandations automatiques de casting selon la tâche.
* Ajouter un aperçu plus visuel du déroulé pédagogique.
* Tester des scénarios à quatre participants avec une scène collective plus flexible.
* Ajouter un export simple de la configuration de rencontre.
