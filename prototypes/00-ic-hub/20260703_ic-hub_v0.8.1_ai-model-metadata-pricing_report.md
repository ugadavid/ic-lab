# IC-Hub V0.8.1 - AI model metadata and pricing

Date: 2026-07-03

## Objectif

La V0.8.1 stabilise le catalogue des modeles IA ajoute en V0.8. Elle n'active aucune generation IA et ne change pas le flux apprenant. Le but est de rendre les modeles synchronises plus lisibles cote administration : description courte, recommandation IC-Lab, usage conseille, modalite, cout qualitatif et champs de prix explicites.

## Fichiers modifies ou crees

- `server/data/ai-model-metadata.json` : metadonnees manuelles non secretes pour quelques modeles de reference.
- `server/ai/modelMetadataEnrichment.js` : enrichissement local des modeles synchronises.
- `server/server.js` : version V0.8.1, store metadata, endpoint admin modeles enrichis, redirections admin/hub V0.8.1.
- `server/ai/openaiModels.js` : preservation des champs enrichis lors d'une resynchronisation OpenAI.
- `server/db/schema.sql` : nouvelles colonnes MariaDB pour metadonnees/prix du catalogue.
- `server/stores/mariadbStore.js` : lecture/ecriture MariaDB des nouveaux champs.
- `server/db/migrate-json-to-mariadb.js` : enrichissement des modeles avant migration.
- `server/db/README.md` : documentation MariaDB mise a jour.
- `server/package.json` : version `0.8.1`.
- `public/admin-0.8.1.html`, `public/admin-0.8.1.css`, `public/admin-0.8.1.js` : vue admin catalogue enrichi.
- `public/hub-0.8.1.html`, `public/hub-0.8.1.css`, `public/hub-0.8.1.js` : copie V0.8.1 du hub.

## Champs ajoutes au catalogue

Les modeles peuvent maintenant porter :

- `short_description` / `shortDescription`;
- `ic_lab_recommendation` / `icLabRecommendation`;
- `pricing_json` / `pricing`;
- `input_price_usd`, `cached_input_price_usd`, `output_price_usd`;
- `pricing_unit`, `pricing_source`, `pricing_last_checked_at`.

Les prix inconnus restent `null`. Le hub n'invente pas de prix : les entrees manuelles utilisent `pricing.source = "manual_review_required"` tant que les tarifs officiels n'ont pas ete verifies.

## Protections ajoutees

- Enrichissement local sans appel reseau et sans exposition de cle API au navigateur.
- Aucune requete OpenAI cote browser : l'admin appelle seulement l'endpoint local `/api/admin/ai/providers/openai/sync-models`.
- Correction du parsing prix : `null` et chaine vide restent `null`, au lieu d'etre transformes en `0`.
- Preservation des metadonnees manuelles lors d'une resynchronisation de modeles.
- Migration MariaDB enrichie avant ecriture en table `ai_models`.

## Interface admin

`admin-0.8.1.html` ajoute :

- filtres par provider, famille, modalite, statut, runtime, cout, recommandation, pertinence Proto06 et prix renseigne/non renseigne;
- affichage de la description courte, recommandation IC-Lab, prix, source tarifaire, date de verification et usage recommande;
- edition admin des champs non secrets.

- Les prix restent a completer manuellement apres verification officielle.
