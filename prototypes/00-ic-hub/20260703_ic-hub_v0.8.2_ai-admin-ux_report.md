# IC-Hub V0.8.2 - AI admin UX

Date: 2026-07-03

## Probleme UX identifie

La V0.8.1 exposait correctement les providers et les modeles IA, mais l'interface admin restait trop technique : champs editables visibles partout, vocabulaire peu explique, synchronisation OpenAI peu rassurante, absence de separation nette entre consultation et edition.

La V0.8.2 ne change pas les capacites IA du Hub. Elle rend l'administration IA plus lisible et plus explicite.

## Changements de vocabulaire

La page `admin-0.8.2.html` ajoute une introduction `Administration des IA` et trois cartes pedagogiques :

- Provider : fournisseur technique comme OpenAI, Anthropic, Google, Mistral ou Ollama.
- Modele : moteur disponible chez un provider, par exemple texte, audio, image, embedding ou temps reel.
- Synchronisation : demande faite par le serveur local pour connaitre les modeles accessibles avec la cle configuree.

Les textes rappellent que la page ne lance aucune IA dans les activites apprenantes.

## Lecture simple / edition avancee

Les providers et modeles sont maintenant affiches sous forme de cartes lisibles.

Par defaut :

- nom, type, statut, badges oui/non;
- description courte;
- recommandation IC-Lab;
- cout qualitatif;
- prix connu ou `Prix non renseigne`;
- runtime autorise oui/non.

L'edition est cachee dans des panneaux `details` :

- `Modifier les informations locales` pour un provider;
- `Modifier la fiche modele` pour un modele;
- `Details techniques` pour les informations avancees.

Le bouton provider est renomme `Enregistrer les informations locales du provider` et precise que l'action ne contacte pas le provider.

## Feedback synchronisation

Le bouton devient :

`Mettre a jour la liste des modeles accessibles avec ma cle OpenAI`

Au clic, le bouton est desactive, un spinner simple apparait et le message indique `Synchronisation en cours...`. A la fin, l'interface affiche :

- nombre de modeles recuperes;
- nombre de modeles au catalogue;
- nombre de modeles `stale`;
- date/heure locale de synchronisation.

Si la cle est absente, le message public devient :

`Impossible de synchroniser : aucune cle OpenAI n'est configuree cote serveur.`

La cle n'est jamais affichee.

## Explication des couts

Un bloc `Comprendre les couts` explique que la synchronisation OpenAI ne fournit pas automatiquement les tarifs detailles et que les prix doivent etre renseignes ou verifies manuellement.

La legende couvre :

- `free`;
- `low`;
- `medium`;
- `high`;
- `very-high`;
- `unknown`.

## Simplification des filtres

Les filtres visibles par defaut sont limites a :

- provider;
- modalite;
- recommandation IC-Lab;
- utilisable Proto06;
- prix connu.

Les filtres famille, statut, runtime et cout sont places dans `Filtres avances`.

## Securite inchangee

- Aucune cle API n'est affichee.
- Aucun contenu `.env` n'est affiche ou logge.
- Le navigateur n'appelle pas OpenAI : il appelle uniquement les endpoints locaux du Hub.
- Aucune generation IA n'est activee dans Prototype 06.
- Les roles teacher/student restent interdits sur les endpoints admin IA.
- `ic_dico` et `docker-compose.yml` ne sont pas modifies.

## Limites restantes

- Le test visuel a ete fait par inspection HTML/API, pas par capture navigateur automatisee.
- La synchronisation OpenAI reelle depend toujours de la cle serveur et du reseau.

- Les modeles ne sont toujours pas actives dans Prototype 06 par cette passe.
