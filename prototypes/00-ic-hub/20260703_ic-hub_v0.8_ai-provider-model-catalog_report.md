# IC-Hub V0.8 - catalogue providers et modeles IA

Date : 2026-07-03

## Objectif

Creer une couche d'administration des providers et modeles IA dans le Hub, sans brancher l'IA au parcours apprenant et sans exposer de secret.

## Pourquoi un catalogue admin

Les AIConfig ne doivent plus dependre uniquement de champs texte libres `provider` / `modelId`.

La V0.8 prepare un catalogue central :

- providers connus ;
- modeles disponibles ;
- modeles autorises ou non pour les enseignants, etudiants et runtimes ;
- cout qualitatif ;
- usage recommande ;
- statut de disponibilite.

Ce catalogue permettra ensuite aux AIConfig de pointer vers un modele valide.

## Provider OpenAI

Le Hub charge explicitement le `.env` place a la racine IC-Lab depuis le serveur uniquement.

Variables prises en compte :

- `OPENAI_API_KEY`
- `OPENAI_ORG_ID`
- `OPENAI_PROJECT_ID`
- `OPENAI_DEFAULT_MODEL`

La cle n'est jamais retournee au client, jamais affichee dans l'admin et jamais loggee.

L'admin voit uniquement :

- cle presente : oui/non ;
- provider activable : oui/non ;
- modele par defaut configure : oui/non.

## Synchronisation OpenAI

Module cree :

- `server/ai/openaiModels.js`

Fonctions :

- `getOpenAiProviderStatus()`
- `listOpenAiModelsFromApi()`
- `syncOpenAiModels()`

La synchronisation appelle `https://api.openai.com/v1/models` cote serveur uniquement.

Comportement :

- recupere les modeles accessibles a la cle locale ;
- upsert dans le catalogue ;
- `source = openai-sync` ;
- met a jour `lastSeenAt` ;
- ne supprime pas brutalement les modeles absents ;
- marque les anciens modeles synchronises non revus comme `stale`.

Verification realisee avec acces reseau autorise : 118 modeles synchronises, 119 modeles au catalogue avec le seed.
