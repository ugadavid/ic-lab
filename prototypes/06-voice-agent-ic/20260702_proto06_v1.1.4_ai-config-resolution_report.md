# Prototype 06 V1.1.4 - resolution read-only de la configuration IA

Date : 2026-07-02

## Objectif

Permettre a Prototype 06 lance depuis le hub de lire `aiConfigId`, `runId` et `launchToken`, puis de recuperer les metadonnees IA/voix associees en lecture seule.

## Fichiers crees

- `index-1.1.4.html`
- `script-1.1.4.js`
- `style-1.1.4.css`

## Changements

- `index-1.1.4.html` reference `style-1.1.4.css` et `script-1.1.4.js`.
- `script-1.1.4.js` lit toujours les parametres URL V1.1.3, avec `aiConfigId`, `runId` et `launchToken`.
- Si `runId`, `launchToken` et `aiConfigId` sont presents, le script appelle :
  - `GET http://127.0.0.1:8790/api/runs/:runId/ai-config`
  - en-tete `X-Launch-Token`
- La reponse est stockee dans l'etat local `hubLaunchState.aiConfig`.
- La carte enseignante affiche :
  - titre ;
  - mode ;
  - provider ;
  - mode/provider voix ;
  - statut ;
  - cout qualitatif ;
  - duree max ;
  - role pedagogique ;
  - avertissements ;
  - mention explicite `IA reelle : non activee`.
- Le debug affiche les champs detailles de la configuration et `runtimeEnabled=false`.
- En cas d'erreur de resolution hub, la rencontre continue et affiche un avertissement non bloquant.

## Verifications

- `node --check script-1.1.4.js` : OK
- backend Proto06 existant sur `8788` : `index-1.1.4.html` servi en `200`
- `script-1.1.4.js` servi en `200`
- `style-1.1.4.css` servi en `200`
- URL de lancement hub testee avec `activityId`, `activitySource`, `courseId`, `assignmentId`, `runId`, `launchToken`, `aiConfigId`
- endpoint hub de resolution teste en JSON et MariaDB
- echec token invalide teste : `401` propre cote hub

## Limites restantes

- Aucun appel IA, aucun provider externe, aucune cle API.
- La resolution depend du hub local `http://127.0.0.1:8790`.
- Si le hub est indisponible, la config IA complete n'est pas affichee, mais l'activite reste utilisable.
