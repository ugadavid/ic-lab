# IC-Lab Hub V0.7 - AIConfig

Date : 2026-07-02

## Objectif

La V0.7 ajoute une premiere couche `AIConfig` au hub IC-Lab pour preparer les futurs modes IA et voix, sans appeler de provider externe et sans stocker de cle API.

Prototype 06 sert de premier cas d'usage, mais il n'est pas modifie dans cette passe.

## Pourquoi maintenant

Le hub sait deja gerer comptes, cours, assignations, traces, visibilite, provenance, MariaDB et procedures stockees. Avant de brancher une vraie IA, il devient utile de rendre visibles les choix pedagogiques et economiques : mode sans IA, IA texte future, voix navigateur, voix API future, temps reel futur, multi-agents, durees maximales et risques.

## Structure

Store JSON :

- `server/data/ai-configs.json`

Tables MariaDB :

- `ai_configs`
- `activity_ai_config`

Champs principaux :

- identite : `id`, `title`, `description`, `prototypeId`;
- moteur : `mode`, `provider`, `modelId`;
- voix : `voiceMode`, `voiceProvider`;
- cout : `estimatedCostLevel`, `estimatedCostNotes`, `costVisibleToTeacher`;
- limites : `maxDurationSeconds`, `requiresApiKey`;
- pedagogie : `languagePolicy`, `pedagogicalRole`, `allowParticipantAgents`, `allowTutorAgent`, `allowObserverAgent`;
- etat : `status`, `warnings`, `createdAt`, `updatedAt`.

## Profils seedes

- `aicfg_proto06_scripted_browser_voice` : gratuit, scenarise, voix navigateur, actif.
- `aicfg_proto06_llm_text_browser_voice` : IA texte future, voix navigateur, prototype.
- `aicfg_proto06_llm_text_api_voice` : IA texte future, voix API future, prevu.
- `aicfg_proto06_realtime_voice_agent` : vocal temps reel futur, tres couteux, experimental.
- `aicfg_proto06_multi_agent_experimental` : multi-agents de recherche, risque pedagogique eleve.

## Endpoints ajoutes

- `GET /api/ai-configs`
- `GET /api/ai-configs/:id`
- `POST /api/ai-configs`
- `PUT /api/ai-configs/:id`
- `GET /api/prototypes/:prototypeId/ai-configs`

Lecture :

- tous les utilisateurs connectes lisent les configs actives;
- teacher/admin lisent toutes les configs.

Ecriture :

- creation/modification reservees a l'admin.

## Interfaces

Admin V0.7 :

- page `admin-0.7.*`;
- section `Configurations IA`;
- affichage titre, prototype, mode, provider, modele, voix, cout, duree, statut, warnings;
- modification admin de statut, description, cout, duree, warnings, provider et modelId;
- aucune cle API demandee.

Teacher V0.7 :

- page `teacher-0.7.*`;
- section `Modes IA disponibles pour Prototype 06`;
- affichage des couts qualitatifs, durees, voix, warnings et statuts;
- mention dans les activites assignees : `Mode actuel : scenarise + voix navigateur`.

## Non branche en V0.7

- aucun appel OpenAI ou autre provider;
- aucune generation de texte;
- aucune generation audio API;
- aucun agent temps reel;
- aucun secret ou cle API;
- Prototype 06 ne consomme pas encore `aiConfigId`.

## Lien futur avec Prototype 06

La V0.7 prepare le champ conceptuel `aiConfigId` et la table `activity_ai_config`, afin qu'une activite puisse plus tard etre associee a un mode IA precis. Pour l'instant, le mode courant affiche reste le mode scenarise + voix navigateur.

## Limites restantes

- `activity_ai_config` est preparatoire et peu utilisee en V0.7.
- Pas de calcul de cout reel.
- Pas de validation fine par provider/mode.
- Pas de branchement Prototype 06.
- La visibilite pedagogique des configs est simple et pourra etre affinee par institution ou cours.
