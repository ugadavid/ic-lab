# IC-Hub V0.9.2 - AI Lab grille d'evaluation non stockante

Date: 2026-07-03

## Objectif

La V0.9.2 ajoute une grille d'evaluation pedagogique manuelle pour comparer les brouillons generes dans l'AI Lab. Elle reste admin-only, locale au navigateur, sans stockage serveur et sans activation dans Prototype 06.

## Criteres d'evaluation

Chaque brouillon de session affiche une section `Evaluation pedagogique` avec sept criteres notes de 1 a 5 :

- clarte pour l'intercomprehension;
- transparence lexicale;
- adequation au scenario;
- naturel de la langue cible;
- prudence pedagogique;
- reutilisabilite dans Prototype 06;
- cout / interet du modele.

Chaque critere possede une aide courte visible dans l'interface.

## Score synthetique local

Le score est calcule cote navigateur par moyenne simple des criteres renseignes.

Badges affiches :

- 4.5 a 5 : excellent candidat;
- 3.5 a 4.49 : candidat interessant;
- 2.5 a 3.49 : a retravailler;
- moins de 2.5 : a eviter;
- aucune note : non evalue.

Le score n'est jamais envoye au serveur.

## Commentaires locaux

Chaque brouillon possede un champ `Commentaire d'evaluation` limite a 500 caracteres. Le commentaire reste dans `sessionDrafts`, en memoire JavaScript seulement. Il disparait au rechargement.

## Favoris de session

Chaque brouillon peut etre marque comme favori de session. Ce favori reste local a la page et n'est pas stocke.

## Synthese de session

La synthese affiche :

- nombre de brouillons generes;
- nombre de brouillons evalues;
- meilleur score;
- personnage du meilleur brouillon;
- modele du meilleur brouillon;
- nombre de favoris;
- rappel de non-stockage.

Deux tris locaux sont disponibles : par score et par personnage.

## Export JSON local

Le bouton `Exporter l'evaluation de session en JSON` genere un fichier cote navigateur avec :

- textes generes;
- metadonnees;
- checks;
- notes;
- commentaires;
- favoris;
- date locale d'export.

L'export n'appelle pas le serveur. Il ne contient pas de secret, token, cle API ou donnee apprenante. Un avertissement visible rappelle de relire le fichier avant partage.

## Absence de stockage serveur

La V0.9.2 ne stocke pas :

- les brouillons en base;
- les evaluations en base;
- les commentaires en base;
- les favoris en base;
- les scores dans `run_events`;
- les textes generes dans `run_events`;
- les brouillons ou evaluations en `localStorage`.

## Absence de branchement Prototype 06

Prototype 06 n'est pas modifie. Le connecteur, le runtime et le parcours apprenant restent scenarises. Aucun brouillon n'est injecte dans une activite.

## Absence de donnees apprenantes

L'AI Lab n'envoie aucun texte apprenant, aucun launchToken et aucune donnee personnelle a OpenAI. Le navigateur n'appelle pas OpenAI directement.

## Limites restantes

- Les scores sont subjectifs et manuels.
- Il n'y a pas encore de grille d'accord inter-evaluateurs.
- L'export JSON est utile pour analyse manuelle, mais doit etre relu avant partage.
- Aucun mecanisme de validation linguistique automatique n'est ajoute.
- Les brouillons restent des objets de recherche, pas des contenus pedagogiques validés.

## Prochaine etape possible

Ajouter une grille de comparaison multi-evaluateurs hors ligne, par import/export local, avant toute reflexion sur une integration dans Prototype 06.
