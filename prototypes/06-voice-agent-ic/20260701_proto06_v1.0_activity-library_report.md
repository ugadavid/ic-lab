# Prototype 06 — Rapport V1.0

## Objectif de la V1.0

La V1.0 ajoute une bibliothèque locale d'activités REPLI4C.

Elle permet à un enseignant de retrouver, filtrer, consulter, lancer, dupliquer, importer et exporter des activités natives ou créées localement.

La bibliothèque devient une mémoire pédagogique locale : elle documente pourquoi une rencontre existe, avec quels personnages, dans quel scénario pédagogique et dans quel but.

## Activités natives et locales

Les activités natives viennent de `activityCatalog`.

Les activités locales viennent de `localStorage`, sous la clé :

```js
proto06CustomActivities
```

Les deux types sont affichés ensemble dans la bibliothèque avec une indication de source : `native` ou `locale`.

## Fonctionnement de la bibliothèque

`library-1.0.html` affiche une carte par activité avec :

* titre ;
* objectif pédagogique ;
* scénario ;
* personnages ;
* langues ;
* thèmes climatiques ;
* durée estimée ;
* auteur ;
* statut ;
* source ;
* tags ;
* dates disponibles.

Actions disponibles :

* `Lancer`
* `Modifier`
* `Dupliquer`
* `Exporter`
* `Supprimer`, uniquement pour les activités locales.

## Recherche et filtres

La recherche texte porte sur :

* titre ;
* objectif ;
* auteur ;
* tags ;
* thèmes ;
* personnages ;
* langues.

Filtres disponibles :

* source ;
* scénario pédagogique ;
* langue ;
* thème climatique ;
* durée ;
* statut.

## Détail d'activité

Le panneau de détail montre :

* titre ;
* objectif ;
* pourquoi cette activité existe ;
* notes enseignantes ;
* consigne ;
* question commune ;
* personnages et langues ;
* thèmes ;
* tags ;
* version ;
* source ;
* dates.

Quand le debug est actif, le JSON de l'activité peut être affiché dans la zone JSON.

## Lancement

Le bouton `Lancer` stocke :

```js
proto06SelectedActivity
proto06SelectedActivitySource
proto06SelectedCharacters
proto06SelectedPedagogicalScenario
```

Puis il ouvre `index-1.0.html`.

## Modification

Le bouton `Modifier` ouvre `admin-1.0.html`.

Pour une activité native, l'administration ouvre une copie modifiable.

Pour une activité locale, elle ouvre l'activité locale existante.

Les clés temporaires utilisées sont :

```js
proto06EditingActivity
proto06EditingActivitySource
```

## Duplication

Le bouton `Dupliquer` crée une copie locale :

* nouvel id local ;
* titre `Copie de ...` ;
* statut `local-draft` ;
* dates mises à jour ;
* sauvegarde dans `proto06CustomActivities`.

## Suppression locale

La suppression est autorisée uniquement pour les activités locales.

Les activités natives du catalogue ne peuvent pas être supprimées.

## Import / export JSON

L'export place le JSON de l'activité dans la zone JSON.

L'import se fait par collage JSON. Validation minimale :

* `id`
* `title`
* `scenarioId`
* `characterIds`

Si un id local existe déjà, un nouvel id local d'import est généré.

## Articulation

La bibliothèque relie :

* `library-1.0.html` pour retrouver et gérer les activités ;
* `admin-1.0.html` pour créer ou modifier une activité ;
* `composer-1.0.html` pour composer manuellement ou charger une activité ;
* `index-1.0.html` pour lancer la rencontre.

## Debug

Le debug existant est conservé.

Dans la bibliothèque, quand `proto06Debug` est actif, la page affiche :

* nombre d'activités natives ;
* nombre d'activités locales ;
* clés `localStorage` utilisées ;
* activité sélectionnée ;
* boutons `Reset rencontre` et `Désactiver debug`.

## Limites restantes

* La bibliothèque reste locale au navigateur.
* Pas de synchronisation entre enseignants.
* Pas encore d'import fichier `.json`.
* Pas encore de téléchargement automatique de fichier JSON.
* Pas encore de validation pédagogique avancée.
