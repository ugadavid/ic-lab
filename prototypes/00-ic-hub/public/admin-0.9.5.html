            Glossaire rapide
            

              
Provider
Fournisseur technique de modeles IA.

              
Modele
Moteur disponible chez un provider.

              
AIConfig
Profil local qui decrit un mode IA/voix possible pour un prototype.

              
Runtime
Contexte dans lequel une activite execute un comportement.

              
Runtime scenarise
Parcours controle, sans generation IA.

              
Runtime generatif
Parcours qui ferait produire du contenu par un modele IA.

              
Sync
Lecture cote serveur de la liste des modeles accessibles.

              
Stale
Modele vu avant mais non retrouve lors d'une synchronisation recente.

              
Prix input
Cout indicatif des tokens envoyes au modele.

              
Prix output
Cout indicatif des tokens produits par le modele.

              
Allowed for runtime
Rend le modele selectionnable pour une future configuration, sans l'activer dans Proto06.

              
Recommended IC-Lab
Marquage local pour guider les choix admin apres verification.

            

          


          

            
Si vous reprenez ce projet

            
Le fichier `.env` doit rester a la racine IC-Lab et ne doit jamais etre committe. Les cles ne sont jamais visibles dans l'admin. La synchronisation OpenAI se fait cote serveur. Les modeles synchronises ne sont pas automatiquement utilises dans les activites. Prototype 06 reste scenarise tant qu'un runtime generatif n'est pas explicitement active. Les prix doivent etre verifies manuellement. En cas de doute, desactiver `allowedForRuntime`.

            
Ouvrir le guide mainteneur IA

            
    
              
  1. Verifier l'etat IA.
    2. 
              
  2. Synchroniser les modeles.
    3. 
              
  3. Choisir 2 ou 3 modeles recommandes.
    4. 
              
  4. Associer une AIConfig.
    5. 
              
  5. Tester cote serveur seulement.
    6. 
              
  6. Brancher le parcours apprenant uniquement apres validation.
    7. 
            

          


          

            

              03
              
Catalogue IA
Providers, synchronisation, couts, recommandations et modeles autorises.

              Catalogue local
            

          

            

              
Provider 

              
Un provider est un fournisseur technique : OpenAI, Anthropic, Google, Mistral, Ollama, etc.

            

            

              
Modele 

              
Un modele est un moteur utilisable chez un provider : modele texte, audio, image, embedding, temps reel, etc.

            

            

              
Synchronisation 

              
La synchronisation OpenAI demande au serveur local quels modeles sont accessibles avec la cle configuree. La cle reste cote serveur et n'est jamais affichee.

            

          


          

            

              
Catalogue OpenAI

              
Contacte OpenAI cote serveur pour lire la liste des modeles. Ne lance aucune generation et n'envoie aucun texte apprenant.

Comprendre les couts

            
Les couts affiches ici sont indicatifs. La synchronisation OpenAI ne fournit pas automatiquement les prix detailles. Les prix doivent etre renseignes ou verifies manuellement a partir des tarifs officiels. Quand le prix n'est pas connu, le Hub affiche "Prix non renseigne".

            

              free aucun cout API
              low cout faible
              medium cout intermediaire
              high cout eleve
              very-high cout tres eleve
              unknown prix non renseigne
