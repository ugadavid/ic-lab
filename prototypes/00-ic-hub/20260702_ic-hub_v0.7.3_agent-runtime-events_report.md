# IC-Hub V0.7.3 - events Agent Runtime et lancement Proto06 V1.2

Date : 2026-07-02

## Objectif

Accompagner Prototype 06 V1.2 en autorisant des traces runtime non sensibles et en mettant a jour l'URL de lancement.

## Fichiers modifies

- `server/server.js`
- `server/data/prototypes.json`

## Changements

- Version serveur passee a `0.7.3`.
- URL de lancement `proto06` :
  - ancienne : `http://127.0.0.1:8788/index-1.1.4.html`
  - nouvelle : `http://127.0.0.1:8788/index-1.2.html`
- Event types autorises :
  - `ai_config_resolved`
  - `agent_runtime_selected`
  - `agent_runtime_fallback`
- Payload runtime filtre dans `compactRunEventPayload`.

## Donnees autorisees dans les events runtime

- `aiConfigId`
- `status`
- `mode`
- `runtimeEnabled`
- `assignedMode`
- `selectedRuntime`
- `fallback`
- `generated`

Les champs existants restent filtres. Le Hub ne stocke pas de cle, de token de lancement, de texte apprenant complet ou de texte genere dans ces nouveaux events.

## MariaDB

Aucun changement de schema ni de procedure stockee n'a ete necessaire.

La migration JSON -> MariaDB existante a ete relancee pour propager l'URL `index-1.2.html` dans la table `prototypes`.
