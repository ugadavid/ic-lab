# Script de démonstration - Informaticaire

Ce script sert à présenter Informaticaire à des enseignants, chercheurs, ingénieurs pédagogiques ou membres d'un réseau IC.

## Phrase d'introduction courte

Informaticaire est un prototype de mémoire communautaire de l'intercompréhension. Il aide à retrouver des ressources, comprendre leur histoire, voir les liens entre projets et acteurs, identifier ce qui risque de disparaître et préparer des contributions vérifiables.

## Version 5 minutes

### 0:00 - 0:40 - Situer le projet

Dire :

> L'objectif n'est pas de remplacer les enseignants, chercheurs ou bibliothécaires. L'objectif est de rendre visibles les ressources, les liens, les risques et les actions possibles.

Montrer :

- l'accueil ;
- les cinq indicateurs ;
- la phrase de mission.

### 0:40 - 1:30 - Montrer la visite guidée

Dire :

> Pour une première découverte, la page propose un parcours en cinq étapes : comprendre, explorer, relier, sauver, contribuer.

Montrer :

- la section `Découvrir Informaticaire en 5 étapes` ;
- cliquer sur une étape pour montrer la surbrillance.

### 1:30 - 2:40 - Jouer la démo Galanet

Dire :

> Je prends Galanet comme exemple parce que la ressource est centrale, historiquement reliée à Galapro, Miriadi et APICAD, et qu'elle pose une vraie question de récupération.

Montrer :

- cliquer sur `Lancer le parcours Galanet` ;
- montrer la recherche remplie ;
- ouvrir la fiche détaillée ;
- signaler les relations, l'état, les actions possibles et la récupération.

### 2:40 - 3:40 - Montrer les usages communautaires

Dire :

> L'outil peut servir à plusieurs publics : enseignants, chercheurs, ingénieurs pédagogiques, réseaux et formation de formateurs.

Montrer :

- `Pour qui ?` ;
- `Je veux...` ;
- un filtre par public dans la bibliothèque.

### 3:40 - 4:30 - Montrer la contribution

Dire :

> La contribution n'est pas envoyée automatiquement. Elle prépare un brouillon local qui doit ensuite être relu humainement.

Montrer :

- formulaire `Contribuer` ;
- type de contribution ;
- fiche concernée ;
- bouton `Préparer la contribution`.

### 4:30 - 5:00 - Conclure par les décisions

Dire :

> La prochaine étape n'est pas seulement technique. Elle est communautaire : qui valide, où héberger, quels droits, quelle sauvegarde ?

Montrer :

- `Ce prototype n'est pas encore...` ;
- `Prochaines décisions communautaires`.

## Version 12 minutes

### 0:00 - 1:15 - Accroche et problème

Dire :

> Beaucoup de ressources d'intercompréhension ont une histoire riche, mais elles sont dispersées : projets, plateformes, corpus, personnes ressources, archives, droits, états d'accès. Informaticaire essaie de transformer cette mémoire dispersée en outil partageable.

Montrer :

- accueil ;
- indicateurs ;
- mission générale.

### 1:15 - 2:30 - Ce que l'outil fait et ne fait pas

Montrer :

- `Pourquoi Informaticaire ?` ;
- `Ce prototype n'est pas encore...`.

Insister sur :

- conserver ;
- documenter ;
- relier ;
- retrouver ;
- rendre réutilisable ;
- préparer la maintenance ;
- préparer la transmission.

### 2:30 - 3:30 - Publics visés

Montrer :

- section `Pour qui ?` ;
- enseignants ;
- chercheurs ;
- ingénieurs pédagogiques ;
- réseaux / associations.

Dire :

> Le même objet peut être utile différemment selon le public : activité à réutiliser, trace historique, problème technique, priorité de sauvegarde.

### 3:30 - 4:40 - Parcours d'usage

Montrer :

- section `Je veux...` ;
- cliquer sur `Je cherche une ressource pédagogique` ;
- cliquer sur `Je veux savoir si une plateforme existe encore`.

Dire :

> Les parcours ne remplacent pas la recherche, ils donnent des portes d'entrée compréhensibles.

### 4:40 - 6:20 - Bibliothèque et fiche détaillée

Montrer :

- recherche ;
- filtres ;
- une fiche ;
- modale détaillée.

Points à commenter :

- à quoi ça sert ;
- état actuel ;
- réutilisation ;
- acteurs liés ;
- relations ;
- traçabilité documentaire en second plan.

### 6:20 - 8:00 - Démo Galanet

Montrer :

- bouton `Lancer le parcours Galanet` ;
- fiche Galanet ;
- relations Galapro, Miriadi, APICAD ;
- récupération ;
- personnes à contacter ;
- todo documentaire.

Dire :

> Cette fiche montre le principe : une ressource n'est pas seulement un nom, c'est une histoire, un réseau, un état d'accès et une action possible.

### 8:00 - 9:15 - Relations et campagne de récupération

Montrer :

- `Relations` ;
- `Ressources à sauver ensemble`.

Dire :

> L'enjeu est de passer de souvenirs dispersés à des actions : retrouver une archive, identifier un contact, documenter les droits, préparer une transmission.

### 9:15 - 10:15 - Contribution et exports

Montrer :

- formulaire de contribution ;
- brouillon JSON ;
- export JSON des fiches ;
- export CSV de la récupération.

Dire :

> Le prototype reste statique, mais il prépare déjà des formats partageables.

### 10:15 - 11:15 - Prudences et décisions

Montrer :

- `Ce prototype n'est pas encore...` ;
- `Prochaines décisions communautaires`.

Dire :

> Avant un partage large, il faut décider qui valide, où héberger, quels droits appliquer et comment sauvegarder durablement.

### 11:15 - 12:00 - Discussion

Poser une ou deux questions aux testeurs.

## Points à montrer absolument

- Les indicateurs de synthèse.
- La visite guidée en cinq étapes.
- La recherche dans la bibliothèque.
- La fiche détaillée Galanet.
- Les relations entre Galanet, Galapro, Miriadi et APICAD.
- Les ressources à sauver ensemble.
- Le formulaire de contribution.
- Les limites publiques.
- Les décisions communautaires.

## Questions à poser aux testeurs

- En deux minutes, comprenez-vous à quoi sert Informaticaire ?
- Quelle section vous semble la plus utile pour votre rôle ?
- Le vocabulaire est-il clair pour quelqu'un qui n'a pas suivi le mémoire ?
- Quelles fiches devraient être validées ou complétées en premier ?
- Qui pourrait confirmer l'état actuel de Galanet, Galapro, Miriadi ou APICAD ?
- Quels droits ou accès doivent être vérifiés avant partage ?
- Le formulaire de contribution donne-t-il envie de signaler une ressource ?
- La version mobile permet-elle une consultation suffisante en réunion ou en déplacement ?
