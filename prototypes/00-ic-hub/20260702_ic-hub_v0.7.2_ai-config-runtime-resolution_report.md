# IC-Hub V0.7.2 - resolution runtime des configurations IA

Date : 2026-07-02

## Objectif

Permettre a Prototype 06 V1.1.4 de recuperer les metadonnees d'une configuration IA associee a une assignation, sans activer d'IA reelle et sans exposer de secret.

## Fichiers modifies

- `server/server.js`
- `server/data/prototypes.json`

## Changements

- Version serveur passee a `0.7.2`.
- URL de lancement `proto06` mise a jour :
  - ancienne : `http://127.0.0.1:8788/index-1.1.3.html`
  - nouvelle : `http://127.0.0.1:8788/index-1.1.4.html`
- CORS minimal enrichi avec l'en-tete `x-launch-token`.
- Ajout de `GET /api/runs/:runId/ai-config`.

## Endpoint runtime

`GET /api/runs/:runId/ai-config` :

- verifie que le run existe ;
- verifie le `launchToken` via `X-Launch-Token` ;
- retrouve l'assignation du run ;
- retrouve `aiConfigId` ;
- retourne uniquement des metadonnees pedagogiques filtrees ;
- force `runtimeEnabled: false` ;
- ne retourne ni token, ni hash, ni secret.

## Limites restantes

- L'endpoint expose uniquement des metadonnees en lecture seule ; il ne lance aucun moteur IA.
- Le `launchToken` reste transmis dans l'URL de lancement existante afin de conserver le flux V0.7.
- Les warnings DB restants concernent des donnees de test historiques : un run sans evenement et des assignations multiples de la meme activite.
