# Prototype 06 — Rapport V0.8

## Objectif de l'itération

La V0.8 ajoute un modèle d'activité enseignante réutilisable via `activityCatalog`.

Jusqu'ici, le prototype composait une rencontre à partir d'un scénario pédagogique et de personnages. La V0.8 permet de charger une activité complète, c'est-à-dire un ensemble cohérent : objectif, scénario, casting, langues, thèmes, consignes et notes enseignantes.

## Pourquoi introduire `activityCatalog`

`activityCatalog` prépare le passage d'un prototype de composition libre vers une bibliothèque d'activités pédagogiques.

Une activité peut être rejouée, discutée, adaptée, documentée et comparée à d'autres activités. Elle donne une unité enseignante au prototype : ce n'est plus seulement une rencontre, mais une situation pédagogique définie.

## Structure d'une activité

Chaque activité contient notamment :

* `id`
* `title`
* `shortTitle`
* `author`
* `version`
* `status`
* `pedagogicalGoal`
* `scenarioId`
* `characterIds`
* `estimatedDuration`
* `targetUsers`
* `languages`
* `climateThemes`
* `instructions`
* `commonQuestion`
* `teacherNotes`
* `whyThisActivity`
* `createdAt`
* `updatedAt`
* `tags`

## Articulation des données

L'activité référence :

* un scénario pédagogique via `scenarioId` ;
* des personnages via `characterIds`.

Le reste continue d'être reconstruit depuis les catalogues existants :

* personnages dans `characterCatalog` ;
* langues dans `languageCatalog` ;
* lieux dans `placeCatalog` ;
* variantes discursives dans `testimonyVariantCatalog`.

## Compositeur

Le compositeur V0.8 ajoute une section `Activités prêtes à tester`.

Chaque carte affiche :

* titre ;
* objectif pédagogique ;
* scénario ;
* personnages ;
* langues ;
* thèmes climatiques ;
* durée estimée ;
* bouton `Charger cette activité`.

Quand une activité est chargée, le compositeur sélectionne automatiquement le scénario et les personnages. L'utilisateur peut ensuite modifier manuellement le scénario ou le casting, ce qui détache la composition de l'activité chargée.

## Stockage local

La V0.8 ajoute :

```js
proto06SelectedActivity
```

Les clés existantes restent utilisées :

```js
proto06SelectedCharacters
proto06SelectedPedagogicalScenario
```

## Rencontre générée

La rencontre V0.8 applique la priorité suivante :

1. si `proto06SelectedActivity` existe, charger l'activité ;
2. sinon, utiliser la sélection manuelle de personnages et scénario.

Quand une activité est active, un bloc discret `Activité enseignante` affiche le titre et l'objectif pédagogique. Le debug expose aussi l'id, les notes, les tags, la version et la date de l'activité.

## Limites restantes

* Les activités ne sont pas encore éditables depuis l'interface.
* Il n'y a pas encore d'import/export d'activité.
* Le compositeur ne vérifie pas encore automatiquement la cohérence entre objectif, langues et casting.
* Le reset local reste global aux clés `proto06*`.

## Prochaines pistes

* Créer une vraie interface d'administration enseignante.
* Ajouter un export JSON d'activité.
* Ajouter des filtres par langue, thème, difficulté ou durée.
* Ajouter des recommandations de casting selon l'objectif pédagogique.
* Préparer une bibliothèque d'activités REPLI4C.
