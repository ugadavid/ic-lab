# IC-Hub V0.9 - Admin OpenAI server lab

Date: 2026-07-03

## Objectif

La V0.9 allume le premier moteur OpenAI uniquement cote serveur et uniquement dans l'administration du Hub. Elle permet a un admin de verifier le runtime, lancer un smoke test minimal et generer de courts brouillons de temoignages IC.

Prototype 06 n'est pas modifie et le parcours apprenant reste scenarise.

## Endpoints ajoutes

- `GET /api/admin/ai/runtime/status`
- `POST /api/admin/ai/runtime/smoke-test`
- `POST /api/admin/ai/runtime/ic-agent-draft`

Tous ces endpoints sont admin-only. Teacher/student recoivent 403 et un token invalide reste 401.

## Modele choisi depuis le catalogue

Le client admin envoie seulement `modelCatalogId`.

Le serveur verifie :

- modele present dans le catalogue enrichi;
- `providerId = openai`;
- statut disponible;
- `allowedForRuntime = true`;
- `providerModelId` present;
- modele non stale/deprecated.

Le frontend ne peut pas passer un modele arbitraire hors catalogue.

## Securite cle

- `.env` est charge cote serveur depuis la racine IC-Lab.
- `OPENAI_API_KEY` n'est jamais retournee au navigateur.
- La cle n'est pas stockee en base.
- La cle n'est pas loggee.
- L'admin voit seulement cle presente oui/non.

## Smoke test

Le smoke test envoie un prompt fixe cote serveur :

`Reponds exactement : IC-Lab OK`

Il utilise l'API Responses avec `store: false`.

Verification realisee avec le modele catalogue `openai_gpt_4_1` / provider `gpt-4.1` :

- `ok: true`
- texte : `IC-Lab OK`
- `stored: false`

## Generation controlee

L'endpoint `ic-agent-draft` accepte seulement des champs whitelistes :

- personnages autorises;
- langues autorisees;
- scenarios autorises;
- theme climat borne;
- `maxWords` borne.

Tests realises :

- Clara en `es-ES`;
- Marco en `it-IT`;
- Ana en `pt-BR`.

Les reponses indiquent :

- `stored: false`;
- `adminOnly: true`;
- `studentTextSent: false`;
- `learnerFlowAffected: false`.

## Absence de branchement Prototype 06

La V0.9 n'a pas modifie Prototype 06. Aucun texte genere n'est injecte dans une activite. Aucune generation IA n'est activee pour les etudiants.

## Absence de donnees apprenantes

Les prompts V0.9 sont controles cote serveur. Aucun texte apprenant, aucun launchToken et aucune donnee personnelle ne sont envoyes a OpenAI.

## store: false

Tous les appels OpenAI V0.9 passent `store: false`. Les textes generes ne sont pas ecrits en base et ne sont pas ajoutes aux `run_events`.
