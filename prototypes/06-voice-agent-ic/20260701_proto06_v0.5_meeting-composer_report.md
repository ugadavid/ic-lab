# Prototype 06 — Rapport V0.5

## Objectif de l'itération

La V0.5 exploite la structuration introduite en V0.4 pour créer un premier compositeur de rencontre REPLI4C.

L'objectif n'est pas de créer une administration complète, mais de tester une question simple : peut-on construire une rencontre d'intercompréhension en combinant librement des personnages, langues et lieux issus d'un catalogue ?

## Point de départ

La V0.5 repart de la V0.4, qui avait séparé les données en catalogues :

* `languageCatalog`
* `placeCatalog`
* `characterCatalog`
* `scenarioCatalog`
* `activeScenarioId`

La logique visuelle, audio et pédagogique du prototype reste dérivée de la rencontre incarnée déjà stabilisée : scène individuelle, scène collective, synthèse vocale, reconnaissance vocale, historique et Observation IC.

## Fonctionnement du compositeur

La page `composer-0.0.5.html` affiche les personnages disponibles depuis `characterCatalog`.

Chaque carte de personnage présente :

* l'image du personnage ;
* son prénom ;
* sa langue, récupérée depuis `languageCatalog` ;
* son lieu et son pays, récupérés depuis `placeCatalog` ;
* son thème climatique ;
* ses tags.

L'utilisateur peut sélectionner de 2 à 4 personnages. La sélection est affichée dans un panneau dédié, avec un aperçu de la rencontre :

* ordre de passage ;
* langues représentées ;
* lieux représentés ;
* thème général ;
* question commune.

Le bouton `Lancer cette rencontre` enregistre les identifiants sélectionnés dans `localStorage` sous la clé `proto06SelectedCharacters`, puis ouvre `index-0.0.5.html`.

## Articulation avec les catalogues V0.4

La fonction principale `buildScenarioFromSelection(selectedCharacterIds)` reconstruit un scénario à partir d'une liste d'identifiants de personnages.

Pour chaque personnage sélectionné, le script récupère :

* le personnage dans `characterCatalog` ;
* sa langue dans `languageCatalog` ;
* son lieu dans `placeCatalog` ;
* son témoignage existant dans `scenarioCatalog`, quand il existe.

Si un futur personnage n'a pas encore de témoignage scénarisé, le prototype génère un texte placeholder clair : `Témoignage à compléter`.

Cette logique évite de réécrire manuellement Clara, Marco et Ana dans le compositeur. Les données restent portées par les catalogues.

## Fonctions principales

Dans `composer-0.0.5.js` :

* `getCatalogItem()` sécurise la récupération d'un objet dans un catalogue.
* `getCharacterBundle()` regroupe personnage, langue et lieu.
* `getScenarioStepForCharacter()` retrouve le témoignage existant d'un personnage.
* `buildScenarioFromSelection()` construit une configuration de rencontre depuis la sélection.
* `renderCharacters()` affiche les cartes de personnages.
* `renderSelection()` affiche les participants choisis.
* `renderPreview()` affiche l'aperçu et l'objet de configuration.

Dans `script-0.0.5.js` :

* `getStoredSelection()` lit la sélection depuis `localStorage`.
* `buildParticipantStepFromSelection()` reconstruit les données d'affichage d'une étape.
* `buildScenarioFromSelection()` reconstruit le scénario actif utilisé par la rencontre.
* `renderCollectiveCharacters()` accepte désormais jusqu'à quatre personnages affichés dans la scène collective.

## Ce qui reste volontairement inchangé

La V0.5 ne cherche pas à produire une nouvelle interface spectaculaire. Elle conserve autant que possible :

* la scène individuelle ;
* la scène collective ;
* les images et fonds existants ;
* les effets subtils de présence ;
* les contrôles audio ;
* la synthèse vocale ;
* la reconnaissance vocale ;
* l'historique ;
* le panneau Observation IC.

## Limites actuelles

* Le catalogue ne contient encore que Clara, Marco et Ana.
* Les témoignages complets n'existent que pour les personnages déjà présents en V0.4.
* La scène collective reste surtout optimisée pour trois participants, même si un quatrième emplacement provisoire est prévu.
* Il n'y a pas encore de création ou d'édition de personnage depuis l'interface.
* Il n'y a pas encore de sélection de scénario parmi plusieurs scénarios.
* La sélection est stockée localement dans le navigateur, sans persistance partagée.
* Le compositeur ne vérifie pas encore l'équilibre pédagogique des langues ou des thèmes.
