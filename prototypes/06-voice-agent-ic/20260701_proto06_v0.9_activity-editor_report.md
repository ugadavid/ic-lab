# Prototype 06 — Rapport V0.9

## Objectif de l'itération

La V0.9 ajoute une première interface locale d'édition d'activités enseignantes.

L'objectif est de montrer qu'un enseignant peut commencer à devenir auteur d'activités REPLI4C, sans backend, sans compte utilisateur et sans administration complète.

## Activité native et activité locale

Une activité native vient de `activityCatalog` dans `data-0.0.9.js`.

Une activité locale est une copie modifiée et sauvegardée dans `localStorage`, sous la clé :

```js
proto06CustomActivities
```

Les activités locales conservent une structure compatible avec `activityCatalog`.

## Champs éditables

La page `admin-0.0.9.html` permet de modifier :

* `title`
* `shortTitle`
* `author`
* `pedagogicalGoal`
* `scenarioId`
* `characterIds`
* `estimatedDuration`
* `targetUsers`
* `climateThemes`
* `instructions`
* `commonQuestion`
* `teacherNotes`
* `whyThisActivity`
* `tags`
* `status`

Elle ne modifie pas les personnages, les images, les lieux, les langues ou les variantes discursives.

## Fonctionnement du formulaire

La colonne gauche liste les activités natives et locales.

Le bouton `Modifier une copie` charge une activité dans le formulaire. Pour une activité native, un nouvel id local est généré.

La colonne centrale contient le formulaire d'édition.

La colonne droite affiche un aperçu : titre, objectif, scénario, personnages, langues, lieux, thèmes, consigne, question, notes et erreurs éventuelles.

## Stockage local

À l'enregistrement, l'activité est ajoutée ou mise à jour dans `proto06CustomActivities`.

Les champs `createdAt` et `updatedAt` sont mis à jour. Le statut par défaut est `local-draft`.

## Export / import JSON

La V0.9 ajoute un export simple : le JSON de l'activité courante est affiché dans la zone d'import/export.

L'import se fait par collage JSON. La validation minimale vérifie :

* `id`
* `title`
* `scenarioId`
* `characterIds`

## Validations ajoutées

Les validations légères portent sur :

* titre obligatoire ;
* scénario obligatoire ;
* objectif pédagogique obligatoire ;
* question commune obligatoire ;
* au moins 2 personnages ;
* maximum 4 personnages.

## Articulation avec le compositeur

Le compositeur V0.9 affiche les activités natives et les activités locales sauvegardées.

Une activité locale peut être chargée comme une activité native. Au lancement, le compositeur stocke :

```js
proto06SelectedActivity
proto06SelectedActivitySource
proto06SelectedCharacters
proto06SelectedPedagogicalScenario
```

## Articulation avec le lecteur

Le lecteur V0.9 applique la priorité :

1. activité native ou locale si `proto06SelectedActivity` existe ;
2. sélection manuelle si aucune activité n'est disponible.

Le debug affiche aussi la source de l'activité, son id, son objectif, ses notes, ses tags/version/date et son JSON.

## Limites restantes

* Pas encore d'édition des personnages.
* Pas encore d'édition des variantes discursives.
* Pas encore d'import fichier `.json`, seulement collage texte.
* Pas encore de bibliothèque partagée.
* Pas encore de validation pédagogique avancée.

## Prochaines pistes

* Ajouter un import fichier JSON.
* Ajouter une duplication explicite d'activité locale.
* Ajouter un export téléchargement.
* Ajouter une édition guidée des variantes discursives.
* Préparer un format d'activité partageable entre enseignants.
