# Prototype 06 V1.1.1 - lancement URL depuis le hub

## Objectif

Cette micro-version permet a Prototype 06 de recevoir une activite assignee par le hub IC-Lab via des parametres URL.

Exemple :

```txt
http://127.0.0.1:8788/index-1.1.1.html?activityId=climateObservations001&activitySource=server
```

## Fichiers crees

- `index-1.1.1.html`
- `script-1.1.1.js`
- `style-1.1.1.css`

Les fichiers V1.1 existants ne sont pas modifies.

## Parametres lus

Au chargement, `script-1.1.1.js` lit :

- `activityId`
- `activitySource`
- `courseId`
- `assignmentId`

Si `activityId` est present, le script initialise :

- `proto06SelectedActivity`
- `proto06SelectedActivitySource`

Quand l'activite est chargee, il deduit et stocke aussi :

- `proto06SelectedCharacters`
- `proto06SelectedPedagogicalScenario`

## Activites serveur

Si `activitySource=server`, le script reutilise la logique V1.1 existante et recupere l'activite via :

```txt
/api/activities/:id
```

L'activite chargee fournit ensuite les personnages et le scenario pedagogique, puis la rencontre se construit normalement.

## Erreurs propres

Si l'API est indisponible ou si l'activite serveur est introuvable, la page affiche une carte claire dans le panneau enseignant avec un lien de retour vers `library-1.1.html`. Le script ne plante pas.

## Debug

Quand `proto06Debug` est actif, le panneau debug affiche :

- les parametres URL detectes ;
- si une activite a ete chargee depuis l'URL ;
- la source ;
- l'endpoint serveur ;
- les erreurs API ou URL.

## Verifications

- `node --check script-1.1.1.js` : OK.
- `GET /api/activities/climateObservations001` : OK.
- `index-1.1.1.html?activityId=climateObservations001&activitySource=server` : HTTP 200.
- Activite cible : `climateObservations001`.
- Scenario deduit : `globalUnderstanding`.
- Personnages deduits : `clara`, `marco`, `ana`.
- Activite inexistante via API : `404` propre.

## Limite restante

Le hub V0.2 genere actuellement des liens vers `index-1.1.html`. Pour exploiter cette micro-version sans modifier Prototype 06 ancien, le hub devra pointer son `launchUrl` vers `index-1.1.1.html` dans une prochaine passe hub, ou generer explicitement cette URL.
