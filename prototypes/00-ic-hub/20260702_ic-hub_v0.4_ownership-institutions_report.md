# IC-Lab Hub V0.4 - Institutions, paternite et provenance

Date : 2026-07-02

## Objectif

La V0.4 ajoute une couche de metadonnees structurantes au hub IC-Lab : institutions, paternite des objets, provenance des activites et visibilite minimale. Cette passe ne met pas encore en place une gestion fine des droits ni une preuve juridique de propriete intellectuelle.

## Pourquoi maintenant

Le hub sait deja gerer les comptes, roles, cours, inscriptions, prototypes, assignations et traces de session. Avant de mutualiser proprement des activites entre cours, enseignants ou institutions, il faut savoir d'ou vient une activite, qui l'a introduite dans le hub, qui en est responsable et dans quel cadre institutionnel elle circule.

## Vocabulaire

- Createur : personne qui cree l'objet dans le hub ou l'enregistre pour la premiere fois (`createdBy`).
- Proprietaire : responsable principal de l'objet dans le hub (`ownerId`).
- Assignateur : personne qui assigne une activite a un cours (`assignedBy`).
- Etudiant realisateur : personne qui lance et realise une activite ; elle apparait dans les traces de run, pas dans la paternite de l'activite.

## Institutions

Fichier : `server/data/institutions.json`

Structure :

- `version`
- `institutions[]`
- `id`
- `name`
- `shortName`
- `type`
- `country`
- `status`
- `createdAt`
- `updatedAt`

Institutions seed :

- `inst_uga` : Universite Grenoble Alpes
- `inst_repli4c` : REPLI4C

L'institution reste optionnelle : `institutionId` peut valoir `null`.

## Activity ownership

Fichier : `server/data/activity-ownership.json`

Ce registre ne copie pas les activites des prototypes. Il enregistre seulement une fiche de provenance/paternite pour une activite referencee par le hub.

Champs principaux :

- `prototypeId`
- `activityId`
- `activitySource`
- `title`
- `ownerId`
- `createdBy`
- `updatedBy`
- `institutionId`
- `visibility`
- `provenance.kind`
- `provenance.sourcePrototype`
- `provenance.sourceActivityId`
- `provenance.importedAt`
- `provenance.registeredBy`

La provenance `external-prototype` signifie que le hub reference une activite exposee par un prototype externe. Ce n'est pas une preuve juridique.

## Migrations effectuees

Au demarrage serveur :

- creation automatique de `institutions.json` si absent ;
- creation automatique de `activity-ownership.json` si absent ;
- ajout doux de `institutionId`, `createdBy`, `updatedBy` aux utilisateurs ;
- ajout doux de `institutionId`, `createdBy`, `updatedBy`, `ownerId`, `visibility` aux cours ;
- ajout doux de `createdBy`, `updatedBy`, `institutionId`, `visibility` aux assignations ;
- creation ou reutilisation d'un record ownership pour les assignations Prototype 06 existantes.

## Endpoints ajoutes

- `GET /api/institutions`
- `GET /api/institutions/:id`
- `POST /api/institutions` admin
- `PUT /api/institutions/:id` admin
- `GET /api/activity-ownership`
- `GET /api/activity-ownership/:id`
- `PUT /api/activity-ownership/:id` admin ou proprietaire

Les endpoints V0.3.1 de run sont conserves.

Pages creees :

- `hub-0.4.html/css/js`
- `admin-0.4.html/css/js`

- visibilite du cours ;

Student V0.4 reste volontairement simple :

- institution du cours ;
- activites disponibles ;
- lancement de run inchange.
- debug optionnel via `localStorage.icHubDebug = "1"` avec institution/visibilite du cours.

## Limites restantes

- Pas de permissions fines : les metadonnees sont affichees et structurees, mais pas encore appliquees comme modele de droits.
- Le registre ownership est local JSON et n'est pas adapte a plusieurs serveurs concurrents.
- La paternite indique une responsabilite dans le hub, pas une certification juridique.
- Les institutions ne sont pas encore reliees a des politiques de partage.
- L'interface ne fournit pas encore de formulaires complets de gestion ownership.

## Prochaines pistes

- Ajouter un vrai modele de visibilite effectif.
- Relier les institutions a des espaces de partage.
- Ajouter un historique des changements de paternite.
- Permettre a l'enseignant de modifier les notes/provenance visibles.
- Ajouter une page de consultation dediee aux activites mutualisables.
