# Prototype 06 V1.1 - connected local backend

## Objectif

La V1.1 prouve que la memoire des activites REPLI4C peut quitter le seul `localStorage` du navigateur pour devenir une memoire locale partagee via une API. Elle ne cree ni comptes, ni roles, ni authentification, ni IA, ni base de donnees complexe.

## Architecture serveur

Un serveur Node natif a ete ajoute dans `server/`. Il sert les fichiers statiques du prototype et expose une API REST sous `/api` sur le port `8788`.

Commande :

```bash
cd prototypes/06-voice-agent-ic/server
npm start
```

## Endpoints

- `GET /api/health` retourne l'etat du service.
- `GET /api/activities` retourne les activites serveur.
- `GET /api/activities/:id` retourne une activite serveur.
- `POST /api/activities` cree une activite serveur.
- `PUT /api/activities/:id` modifie une activite serveur.
- `DELETE /api/activities/:id` supprime une activite serveur.

## Stockage JSON

Les donnees sont stockees dans `server/data/activities.json` :

```json
{
  "version": "1.1",
  "updatedAt": "2026-07-02T00:00:00.000Z",
  "activities": []
}
```

Avant chaque ecriture, le serveur cree une sauvegarde dans `server/data/backups/`, puis ecrit un fichier temporaire avant renommage pour limiter les corruptions.

## Sources d'activites

La V1.1 articule trois sources :

- `catalog` : activites natives de `activityCatalog`.
- `local` : activites stockees dans `proto06CustomActivities`.
- `server` : activites recuperees depuis `/api/activities`.

Les activites natives ne sont jamais supprimees par l'API. Le reset local reste cible sur les cles `proto06*` et ne touche pas `server/data/activities.json`.

## Bibliotheque

`library-1.1.html/js/css` charge les activites natives et locales, tente `GET /api/health`, puis charge les activites serveur si l'API repond. Elle affiche l'etat `Serveur connecte` ou `Serveur indisponible - mode local`, ajoute un filtre `Serveur`, permet de lancer, modifier, dupliquer, exporter, supprimer localement, supprimer cote serveur et envoyer une activite locale au serveur.

## Admin

`admin-1.1.html/js/css` conserve le formulaire V1.0 et ajoute une destination serveur. `Enregistrer localement` continue d'ecrire dans `localStorage`. `Enregistrer sur le serveur` utilise `POST /api/activities` pour une nouvelle activite et `PUT /api/activities/:id` pour une activite serveur deja existante.

## Compositeur

`composer-1.1.html/js/css` charge les activites natives, locales et serveur. Au lancement, il stocke :

- `proto06SelectedActivity`
- `proto06SelectedActivitySource`
- `proto06SelectedCharacters`
- `proto06SelectedPedagogicalScenario`

La source vaut `catalog`, `local` ou `server`.

## Lecteur

`index-1.1.html`, `script-1.1.js` et `style-1.1.css` conservent le moteur de rencontre V1.0. Si la source selectionnee est `server`, le lecteur recupere l'activite via `/api/activities/:id`. Si l'API est indisponible, il affiche une carte claire avec un lien de retour bibliotheque et continue sans planter.

## Debug

`localStorage.setItem("proto06Debug", "1")` et `window.proto06Debug(true)` restent disponibles. Le debug affiche maintenant l'etat serveur, la source de l'activite, l'id, l'endpoint serveur et les erreurs API. La bibliotheque et le compositeur ajoutent aussi une action de rechargement des activites serveur.

## Fallback

Si le serveur ne repond pas, les pages continuent avec les activites natives et locales. La sauvegarde locale reste disponible. Les activites serveur ne sont pas confondues avec les activites natives.

## Limites restantes

- Le stockage JSON local reste mono-processus et minimal.
- Il n'y a pas de verrouillage avance ni de resolution de conflits.
- Les appels serveur frontend restent volontairement simples.
- Le lecteur utilise une recuperation synchrone courte pour charger l'activite serveur avant de construire la rencontre.

## Pistes suivantes

Les prochaines versions pourraient introduire des comptes, des roles, des cours, des collections partagees, une vraie base de donnees, une API multiutilisateurs, des permissions par enseignant et une synchronisation entre plusieurs postes.
