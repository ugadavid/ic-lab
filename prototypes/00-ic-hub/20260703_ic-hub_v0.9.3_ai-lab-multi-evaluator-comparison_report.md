# IC-Hub V0.9.3 - AI Lab comparaison multi-evaluateurs hors ligne

Date: 2026-07-03

## Objectif

La V0.9.3 permet a plusieurs evaluateurs de produire chacun un export JSON local depuis l'AI Lab, puis d'importer plusieurs exports dans l'admin pour comparer les evaluations hors ligne. Rien n'est uploade, rien n'est stocke cote serveur, et Prototype 06 reste inchangĂ©.

## Fichiers ajoutes ou modifies

- `server/server.js`
- `server/package.json`
- `server/package-lock.json`
- `public/admin-0.9.3.html`
- `public/admin-0.9.3.css`
- `public/admin-0.9.3.js`
- `public/hub-0.9.3.html`
- `public/hub-0.9.3.css`
- `public/hub-0.9.3.js`
- `docs/ai-admin-maintainer-guide.md`

La V0.9.2 reste disponible dans ses fichiers existants.

## Nom ou code evaluateur

L'admin ajoute un champ local `Nom ou code evaluateur`, limite a 60 caracteres. Il est facultatif mais recommande et n'est jamais envoye au serveur.

## Export JSON V0.9.3

L'export de session contient maintenant :

- `schema`;
- `schemaVersion`;
- `rubricVersion`;
- `exportedAt`;
- `evaluator.label`;
- `drafts`;
- `ratings`;
- `score`;
- `comment`;
- `favorite`;
- `safety`.

Il n'exporte pas de cle API, token, launchToken, email, userId ou donnee apprenante.

## Import multi-evaluateurs

La section `Comparaison multi-evaluateurs` permet d'importer plusieurs fichiers JSON :

- import cote navigateur uniquement;
- limite 2 Mo par fichier;
- validation de schema;
- messages d'erreur lisibles;
- aucun upload serveur;
- rendu en DOM local, avec insertion via `textContent` pour le tableau.

## Comparaison des evaluations

L'admin calcule localement :

- nombre de fichiers importes;
- nombre d'evaluateurs;
- nombre de brouillons compares;
- moyenne globale;
- meilleur brouillon moyen;
- brouillons avec fort desaccord.

Le tableau de comparaison affiche :

- brouillon;
- personnage;
- langue;
- scenario;
- theme;
- modele;
- nombre d'evaluateurs;
- score moyen;
- score min;
- score max;
- ecart;
- favoris;
- recommandation.

## Details par brouillon

Chaque brouillon compare affiche :

- texte;
- scores par evaluateur;
- commentaires;
- moyenne par critere;
- criteres faibles;
- criteres controverses.

## Export local de comparaison

L'export de comparaison genere un JSON cote navigateur avec :

- agregats;
- moyennes;
- desaccords;
- recommandations;
- date locale d'export;
- fichiers/evaluateurs importes.

Il ne contacte aucun endpoint serveur.

## Absence de stockage serveur

La V0.9.3 ne stocke pas :

- brouillons en base;
- evaluations en base;
- imports en base;
- comparaison en base;
- texte genere dans `run_events`;
- score dans `run_events`;
- brouillons/evaluations/imports dans `localStorage`.

## Absence de branchement Prototype 06

Prototype 06 n'est pas modifie. Le connecteur, le runtime et le parcours apprenant restent scenarises. Aucun brouillon n'est injecte dans une activite.

## Limites restantes

- La comparaison repose sur des exports manuels.
- Il n'y a pas encore de rapprochement avance si deux brouillons proches ont des textes differents.
- Les seuils de desaccord sont simples.
- L'export de comparaison doit etre relu avant partage.

## Prochaine etape possible

Ajouter une aide a la consolidation pedagogique hors ligne : selectionner un brouillon candidat et produire une fiche de relecture humaine, toujours sans branchement Prototype 06.
